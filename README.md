# Teaching Skills

> A collection of reusable teaching-style skill documents for AI tutors, Claude Skills, prompt engineering, and human-friendly learning systems.

**Teaching Skills** is a repository for storing structured teaching methods that help students understand, learn, and remember concepts more effectively. Each file is written like a reusable skill document that can be copied into an AI tutor, adapted into Claude Skills, or used as a teaching framework for technical subjects like COA, DSA, OS, DBMS, and more.

---

## Why This Repo Exists

Most teaching prompts only say:

```text
Explain this topic simply.
```

But great teaching needs structure.

This repo stores teaching frameworks that define **how** something should be taught, not just **what** should be taught.

The goal is to create teaching styles that make learning feel:

* visual
* simple
* step-by-step
* memorable
* exam-friendly
* beginner-friendly
* recall-focused
* neuroscience-aware

## Available Skills

### 1. Abdul Bari-Style Teaching Doc

File:

```text
abdul_bari_style_teaching_doc.md
```

A concept-first whiteboard teaching style inspired by slow, visual, engineering-focused explanations.

Best for:

* computer architecture
* algorithms
* operating systems
* data structures
* exam preparation
* beginner-friendly technical explanations

Core pattern:

```text
Problem → Real-life analogy → Diagram → Step-by-step explanation → Technical definition → Exam answer
```

Example teaching rhythm:

```text
Imagine you are studying.

Study...
Study...
Study...

Suddenly:
Phone rings.

That phone call is like an interrupt.
```

---

### 2. Neuroscience-Based Teaching Skill

File:

```text
neuroscience_based_teaching_skill_md.md
```

A teaching framework based on learning science and memory methods.

Best for:

* deep understanding
* long-term memory
* active recall
* spaced revision
* exam writing
* concept mastery
* AI tutor prompt design

Core pattern:

```text
Scene → Problem → Map → Chunk → Explain → Recall → Correct → Compress → Exam
```

Main learning loop:

```text
Understand → Encode → Recall → Retain
```

## How to Use

Copy any skill document and paste it into your AI tutor prompt, Claude Skill, custom GPT instructions, or learning workflow.

Example:

```text
Use the teaching style from abdul_bari_style_teaching_doc.md.
Teach me Interrupts in COA using that style.
```

Or:

```text
Use NeuroLearn Mode from neuroscience_based_teaching_skill_md.md.
Teach me RAM, ROM, and their types so I can understand, remember, and write answers in exams.
```

## Master Prompt

Use this prompt to activate the full teaching system:

```text
You are my AI tutor.
Teach me the topic using the teaching skills from this repository.

Use:
- concept-first explanation
- real-life analogy
- memory map
- step-by-step breakdown
- visual diagrams
- active recall
- common mistake correction
- exam-safe answer
- revision hooks

Do not start with heavy textbook definitions.
First make me understand why the topic exists.
Then teach it slowly and visually.
```

## Example Output Style

Instead of this:

```text
An interrupt is a signal generated by hardware or software to get CPU attention.
```

The teaching skill prefers this:

```text
First understand the problem.

CPU is working.
Working...
Working...
Working...

Suddenly, a keyboard needs attention.

Instead of CPU checking again and again,
the keyboard sends a signal.

That signal is called an interrupt.
```

This style makes the concept easier to understand before memorizing the definition.

## Skill Design Principles

Every teaching skill in this repo should follow these principles:

### 1. Teach the Problem First

Before teaching the concept, explain why the concept exists.

### 2. Use Analogies

Map abstract ideas to real-life situations.

### 3. Use Memory Maps

Show the full structure before going into details.

### 4. Use Active Recall

Make the learner retrieve information, not just reread it.

### 5. Correct Common Confusions

Mention what students usually misunderstand.

### 6. Give Exam-Safe Answers

Separate simple intuition from final exam-ready wording.
